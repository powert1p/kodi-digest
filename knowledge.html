Архитектурные паттерны

1. Leader-Worker (Координатор → Исполнители)

  • Что: Один агент-лидер декомпозирует задачу и раздаёт подзадачи worker-агентам
  • Когда: Задача декомпозируется на 3+ независимых кусков
  • Пример: Anthropic C-compiler — 16 агентов, 2000 сессий, $20K
  • В стеке: Claude Code Agent Teams

2. Swarm (Самоорганизующийся рой)

  • Что: Пул задач → воркеры сами берут следующую. Нет центрального координатора
  • Когда: Исследовательские задачи, где scope неизвестен
  • В стеке: SubAgent-driven development через superpowers skill

3. Pipeline (Конвейер)

  • Что: A → B → C. Каждый агент принимает артефакт предыдущего
  • Когда: Чёткий процесс: PM → Architect → Implementer → Tester → Reviewer
  • В стеке: Superpowers skills chain (brainstorming → plans → TDD → verify)

4. Competitive (Конкурирующие гипотезы)

  • Что: Несколько агентов решают одну задачу разными способами, лучшее побеждает
  • Когда: Дебаг сложных проблем с множеством возможных причин
  • В стеке: Agent Teams team-debug preset

5. Watchdog (Наблюдатель)

  • Что: Отдельный агент мониторит работу других и вмешивается при проблемах
  • Когда: Долгие задачи, где нужен quality gate
  • В стеке: superpowers:requesting-code-review

Практические паттерны

  • File Ownership: Каждый файл = ровно один агент. Никогда два параллельно
  • Think-then-Critique: Черновик → Критика → Ревизия (видимый цикл)
  • TDD в агентах: RED → GREEN → REFACTOR per task, никогда не удалять тесты
  • Error Recovery: 3 попытки → WebSearch → эскалация

Тренды 2026

  • Shift от кода к оркестрации агентов
  • Активная human-agent коллаборация (60% с AI, 0-20% полностью делегируется)
  • Multi-agent coordination mastery
  • Ключевой сдвиг: «Монолитные агенты → оркестрированные команды специализированных агентов»
