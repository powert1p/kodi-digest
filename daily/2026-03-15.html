Linux Foundation учредил AAIF — MCP как стандарт
Agentic AI Foundation (AAIF) с MCP, goose, AGENTS.md как основными проектами. MCP roadmap 2026: горизонтальное масштабирование без state, enterprise auth/SSO/audit, .well-known для discovery. Уже 10,000+ MCP серверов в экосистеме. Рядом: OpenAI Agents SDK, Google ADK — всё сходится к MCP.
Подробнее ▶

Что такое AAIF: Linux Foundation создал фонд специально для агентских AI стандартов. MCP (Model Context Protocol) стал де-факто стандартом для подключения AI к внешним инструментам. AAIF легитимизирует это на уровне индустрии — как CNCF для Kubernetes.

Roadmap 2026: Горизонтальное масштабирование (несколько MCP серверов без общего state) решает проблему, с которой сталкивается OpenClaw — когда один MCP сервер обслуживает несколько агентов одновременно. Enterprise auth/SSO — для корпоративных клиентов. .well-known для discovery — автоматический поиск MCP серверов по домену.

Конкуренты: OpenAI Agents SDK и Google ADK технически совместимы с MCP. Это не война стандартов — все сходятся к одному протоколу. Для разработчика это значит: инвестиции в MCP не обесценятся.

Для AiPlus: Если в будущем AiPlus захочет предлагать AI-powered обучение — MCP позволит подключить AI-агентов к существующей инфраструктуре (amoCRM, Sipuni, учебные материалы) через стандартный протокол.

Одно действие
Кодить с MCP как стандартом — AAIF даёт легитимность. Следить за transport improvements для масштабирования OpenClaw

Cursor Automations — агенты с триггерами внутри IDE
Cursor запустил Automations — агенты, которые запускаются автоматически по триггерам (новый code, Slack-сообщение, таймер). Главный тренд марта: переход с single-prompt agents на long-running autonomous workflows с execution loops. Рынок agentic AI: $9.14B в 2026 → $139B к 2034 (+40.5% CAGR).
Подробнее ▶

Как работает: В Cursor создаёшь Automation — описываешь триггер (PR создан, тест упал, Slack-сообщение) и действие (ревью кода, фикс, ответ). Агент работает в фоне, не нужно каждый раз давать prompt вручную.

Отличие от Claude Code: Claude Code + Conductor работают по запросу (ты даёшь задачу → агент делает). Cursor Automations — event-driven: агент слушает события и реагирует сам. OpenClaw уже делает нечто подобное через heartbeats, но Cursor формализовал это в UI.

Тренд "autonomous workflows": Вместо одного промпта → одного ответа, агенты работают в цикле: plan → execute → observe → re-plan. Conductor уже так работает с worktrees, но без триггеров.

Рынок $139B к 2034: Это значит, что навыки работы с агентами (которые Есет уже развивает) будут востребованы. AiPlus может добавить курсы по AI-агентам для корп.клиентов.

Одно действие
Посмотреть Cursor Automations как reference для event-driven agents в OpenClaw — webhook-triggered workflows

Альтернатива: Make.com и Zapier тоже выпустили MCP серверы, но n8n — open source и self-hosted. Для OpenClaw на домашнем Mac это важно: данные не уходят в облако.

📊 MEASURE
Attribution изменилась: click-only vs engage-through
Meta разделила атрибуцию: клики считают только link-clicks, остальное → engage-through. Порог просмотра видео упал с 10s до 5s. Кампании с ROAS 3.5x могут упасть в отчётах на 20-30% — это арифметика, не реальное падение.
Подробнее ▶

Что изменилось в атрибуции: Раньше один тип "click" включал всё — и реальные переходы по ссылке, и "engagement clicks" (лайк, коммент, share). Теперь это разделено: click = только переход на сайт, engage-through = всё остальное.

Почему ROAS "упадёт": Если ты мерил ROAS по "click attribution", теперь часть конверсий переедет в "engage-through". Визуально в отчёте ROAS упадёт на 20-30%, но реальных конверсий столько же. Не паникуй — пересчитай.

Видео 5s vs 10s: Раньше "просмотр" считался с 10 секунд, теперь с 5. Это означает: больше "просмотров" в отчётах, но качество просмотра ниже. Для образовательного контента AiPlus (длинные ролики) — метрика видео-просмотров раздуется, ориентируйся на ThruPlay (полный просмотр).

Действие: Ads Manager → столбцы отчёта → добавить "engage-through conversions" → сложить с click conversions → сравнить с прошлым месяцем. Если сумма ≈ та же — всё ок, просто новая разбивка.

Одно действие
Ads Manager → Измерение → обновить атрибуцию на click+engage-through → пересчитать ROAS за последний месяц

⚙️ MCP
n8n MCP: автоматизировать 1 DevOps workflow
n8n MCP позволяет агентам вызывать сложные workflow. Для Conductor/OpenClaw — можно делегировать рутинные задачи (деплой, уведомления, обновление данных) через один MCP вызов.
Подробнее ▶

Конкретный use-case: Сейчас деплой дайджеста = exec "cd ~/kodi-digest && git add . && git commit && git push". С n8n MCP: один вызов → n8n делает: git push + проверка GitHub Pages + уведомление в Telegram + запись в лог.

Установка: npm install -g n8n → n8n start → настроить MCP server → добавить в OpenClaw config. Self-hosted на Mac, данные не уходят наружу.

Время инвестиции: ~2 часа на настройку первого workflow. Окупается при ежедневном использовании за неделю.

Действие
Установить n8n MCP server → подключить к OpenClaw → автоматизировать одну рутину (например, деплой дайджеста)
